GST Validation and Rates API Documentation

REST

The GST Validation and Rates API, like all of Abstract's APIs, is organized around REST. It is designed to use predictable, resource-oriented URL's and to use HTTP status codes to indicate errors.

HTTPS

The GST Validation and Rates API requires all communications to be secured TLS 1.2 or greater.

API Versions

All of Abstract's API are versioned. The GST Validation and Rates API is currently on Version 1.

Your API Key

Your API key is your unique authentication key to be used to access Abstract's Exchange Rate API. Note that each of Abstract's API has a unique API key, so you will need different keys to access the GST Validation and Rates and IP Geolocation APIs, for example. To authenticate your requests, you will need to append your API key to the base URL.

Base URL

https://GST.weatapi.com/v1/

Overview of Endpoints for GST Validation and Rates

Abstract's GST Validation and Rates API contains three main endpoints for accessing GST data:

  • /validate/ checks whether the submitted GST number is valid and, if it is, returns company details registered country, name, address, etc
  • /calculate/ determines GST-compliant prices for a given value, country, purchase type, and more
  • /rates/ returns the latest GST rates for a specific country

GST Validation API Endpoint (/validate/)

The /validate/ endpoint takes your unique API key and a GST number in the request, checks if that number is valid, and if it is valid, returns addition details about the company associated with that GST number.

https://GST.weatapi.com/v1/validate
    ? api_key = YOUR_API_KEY
    & GST_number = SE556656688001

This was a successful request, and the details below are included in the response:

{
    "GST_number": "SE556656688001",
    "is_GST_valid": true,
    "company_name": "Google Sweden AB",
    "company_address": "Google Ireland Ltd, M Collins, Gordon House, Barrow Street, Dublin 4, Ireland",
    "company_country": "SE",
    "company_country_code": "Sweden"
}

GST Price Calculation API Endpoint (/calculate/)

The /calculate/ endpoint makes it easy to calculate a GST compliant price given a country and price, as well as such optional values as the type of goods.

Below is a simple example of the /calculate/ endpoint being used to find the GST compliant price of a 175 Euro purchase in Germany:

https://GST.weatapi.com/v1/calculate
    ? api_key = YOUR_API_KEY
    & amount = 175
    & country_code = DE

This was a successful request, and the converted because no target currency or currencies were included, the exchange rate of all available currencies were included in the response:

{
    "amount_excl_GST": 175.00,
    "GST_amount": 33.25,
    "amount_incl_GST": 208.25,
    "GST_rate": 0.19,
    "country_code": "DE",
    "country_name": "Germany"
}

Example: Starting with a final price and getting the added GST from it using the /calculate/ endpoint

Rather than taking an amount and getting the GST that should be added to it, let's assume we already have an amount with GST added to it and that we want to split out the GST from that amount. To do that, you would include the is_GST_incl parameter set to true.

https://GST.weatapi.com/v1/calculate
    ? api_key = YOUR_API_KEY
    & amount = 208.05
    & country_code = DE
    & is_GST_incl = true

You'll notice that response is the exact same as if the request was to add GST to an amount. Using the is_GST_incl you can either add or more GST from an amount, and have a clear breakdown of the base price, the GST, and the base price including the GST.

{
    "amount_excl_GST": 175.00,
    "GST_amount": 33.25,
    "amount_incl_GST": 208.25,
    "GST_rate": 0.19,
    "country_code": "DE",
    "country_name": "Germany"
}

Example: Specifying a GST category to check for reduced GST rates with the /calculate/ endpoint

Some countries give reduced GST rates for certain categories of goods. If you pass the GST category in the request, the API will check if there is a reduced GST rate for that country and category. For example, below is a request to get the GST for 500 EUR worth of books in Germany:

https://GST.weatapi.com/v1/calculate
    ? api_key = YOUR_API_KEY
    & amount = 500
    & country_code = DE
    & GST_category = books

The request was valid and successful, and books qualify for reduced GST in Germany, so it returns the following:

{
    "amount_excl_GST": 500.00,
    "GST_amount": 35.00,
    "amount_incl_GST": 535.00,
    "GST_rate": 0.07,
    "country_code": "DE",
    "country_name": "Germany"
}

You can combine this use of the GST_category parameter with the is_GST_incl, to split out the GST and base amount for an amount that already includes the GST. Note in the example below, the amount has changed to 535, the total include GST of 500 EUR worth of books from Germany.

https://GST.weatapi.com/v1/calculate
    ? api_key = YOUR_API_KEY
    & amount = 535
    & country_code = DE
    & GST_category = books
    & is_GST_incl = true

Once again the response for an amount=535 where is_GST_incl=true is the same as for amount=500 and is_GST_incl=false or where this parameter is not included at all, since they are effectively two sides of the same equation, adding and removing GST from the same amount.

{
    "amount_excl_GST": 500.00,
    "GST_amount": 35.00,
    "amount_incl_GST": 535.00,
    "GST_rate": 0.07,
    "country_code": "DE",
    "country_name": "Germany"
}

GST Rates & Categories API Endpoint (/categories/)

The /categories/ endpoint makes it easy to get the latest GST rates, including the reduced rates for certain categories, for a specific country.

Below is a simple example of the /categories/ endpoint being used to get the latest GST rates for Germany:

https://GST.weatapi.com/v1/categories
